pawsSDB::__construct

Description

You need to instantiate an instance of a pawsSDB object in order to use the class functions. The constructor has no arguments.

If you will be accessing more than one domain at once (e.g., getting attributes from 'domainA', then perhaps depending on the results, some from 'domainB', and again back to 'domainA', you might want to create more than one instance of a pawsDB object. This will allow you to avoid repeated calls to setDomain. And if you are doing queries on multiple domains, then you must have separate instances for getNextQueryItemName to work properly.

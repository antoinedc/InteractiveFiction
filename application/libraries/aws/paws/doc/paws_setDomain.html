pawsSDB::setDomain($domainName)

Description

All operations that get, put, or query data in AWS SimpleDB operate on a specific domain name. It is most likely that you will perform several operations on a single domain at a time. Rather than requiring every single data manipulation call to require a domain name parameter, the domain name is instead saved in the pawsSDB class instance using the setDomain function to change the name as needed.

Most of the time, this will be exactly what you want to do. Typically, you can keep a single instance of the pawSDB object and use setDomain to change the domain being used.

Sometimes, however, you may want to work with several domains simultaneously. In that case, you can create separate instances of pawsSDB objects to use on individual domains. Either way, using this approach provides the most flexibility while simplifying your code.

Note that neither calling setDomain nor creating a new pawsSDB instance cause transactions with SimpleDB, so you don't need to worry about using either approach causing performance issues.

Parameters

NameDescriptionRequired?
$domainName

The name of the domain to set.

Type: String

 Yes

paws: A High Level PHP Library for Amazon Web Services

Programming Tips

This section contains some tips on how to do common things with SimpleDB and pawsSDB. For now,the tips are here in random order, each with its own heading.

Use the itemName as your key

In traditional data bases, it is common to have one field as the key field which is then used as the main way to access data in the table. (Often the keys are in some sorted order, which isn't currently an option is SimpleDB, but in reality, the order is often irrelevant, just like associative arrays in PHP.) In SimpleDB, you can used the key as the itemName. For example, if you had a data base with unique user names, you can use the username as the itemName to create items directly, and don't need an attribute to hold the username. SimpleDB is optimized to access by itemName, and it will be more efficient to use the itemName as the key rather than performing a query on an attribute name. (But you must have at least one attribute value associated with the item - SimpleDB doesn't allow empty items.)

How to delete an entire item

You can delete an entire item by using deleteAttributes with the itemName only: $mySDB->deleteAttributes($itemName);. (Perhaps pawSDB should have deleteItem as a convenience function? Feedback appreciated.)

How to check if an item exists

You can see if a item with a specific name (e.g., key, see above) exists using getAllAttributes or getAttributeValues. If you know an item will always have an attribute with a specific name, then using getAttributeValues will be slightly more efficient:   if ($mySDB->getAttributeValues('user97','userlevel') != NULL) { // item user97 exists... }. Remember that if you just created or deleted the item, this test may not return the correct results right away because of eventual consistency.
